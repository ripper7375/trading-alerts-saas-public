The failing job is caused by Jest encountering "unexpected token" errors when running several test suites on TypeScript/TSX files. This typically means Jest is not correctly set up to transform modern JavaScript/TypeScript syntax used in your components or tests.

**Solution:**

1. **Ensure You Have Babel or ts-jest Configured for .tsx/.ts Files**
   - For Babel: Install the required dependencies:
     ```
     npm install --save-dev @babel/preset-env @babel/preset-react @babel/preset-typescript babel-jest
     ```
   - Ensure your Jest configuration (`jest.config.js` or `jest.config.ts`) includes Babel or ts-jest for transforming TypeScript/TSX:

     ```js
     // jest.config.js
     module.exports = {
       preset: 'ts-jest', // or use 'jest-expo' for Expo projects
       transform: {
         '^.+\\.(ts|tsx)$': 'ts-jest',
         '^.+\\.(js|jsx)$': 'babel-jest',
       },
       // Add if using Babel:
       // "babel-jest" config should point to .babelrc or babel.config.js
     };
     ```

2. **Or, if using Babel, your babel.config.js should include:**

   ```js
   module.exports = {
     presets: [
       '@babel/preset-env',
       '@babel/preset-react',
       '@babel/preset-typescript',
     ],
   };
   ```

3. **Verify file extensions in testMatch/testRegex**
   Make sure Jest is set to find `.test.ts` and `.test.tsx` files, e.g.:

   ```js
   testMatch: [
     '**/__tests__/**/*.[jt]s?(x)',
     '**/?(*.)+(spec|test).[tj]s?(x)',
   ],
   ```

4. **Check for missing dependency (@types/jest, jest-environment-jsdom)**
   Some projects need these for TypeScript+Jest:
   ```
   npm install --save-dev @types/jest jest-environment-jsdom
   ```

**Summary:** Update your `jest.config.js` to use `ts-jest` or ensure you have the correct Babel presets and configuration. Most often, adding `ts-jest` as the transformer for TS/TSX files resolves "unexpected token" errors.
