The failures are due to two main issues:

1. Use of 'tierConfig' without undefined checks:
   In both app/(dashboard)/dashboard/page.tsx and app/(dashboard)/settings/billing/page.tsx, tierConfig is derived like this:

   ```tsx
   const tierConfig = TIER_CONFIG[userTier];
   ```

   TypeScript error TS18048 is raised when you access properties (e.g., tierConfig.maxAlerts) without checking if tierConfig is undefined. This can occur if userTier is a value not present in TIER_CONFIG. Add a fallback or conditional rendering where you use tierConfig. Example fix:

   ```tsx
   // Before using tierConfig's properties:
   if (!tierConfig) {
     // Handle gracefully, e.g., render an error or default values
     return <div>Invalid user tier</div>;
   }
   ```

   Or, for direct property access:

   ```tsx
   value={`${alertCount}/${tierConfig?.maxAlerts ?? 0}`}
   max={tierConfig?.maxAlerts ?? 0}
   ```

2. Cannot find name 'Tier' (TS2304) in types/tier.ts:
   The Tier type is re-exported from '@/lib/tier-config', but TS can't find it. This typically means the actual file ('lib/tier-config.ts') is missing, misnamed, or not exported as expected.
   - Ensure that lib/tier-config.ts exists and exports Tier:
     ```typescript
     // lib/tier-config.ts
     export type Tier = 'FREE' | 'PRO';
     // ... other exports
     ```
   - If you're using a non-standard TypeScript path alias ('@/lib/...'), confirm your tsconfig.json has the correct baseUrl and paths settings.

Summary of changes needed:

- Add a check for tierConfig before accessing its properties or use optional chaining/defaults.
- Make sure Tier is properly exported from lib/tier-config.ts and that the path alias is correctly configured.

Example dashboard/page.tsx usage fix:

```tsx
const tierConfig = TIER_CONFIG[userTier];
if (!tierConfig) {
  // fallback logic (show error or default)
  return <div>Invalid tier</div>;
}
// ...safe to access tierConfig.maxAlerts, etc.
```

or update usages with optional chaining:

```tsx
value={`${alertCount}/${tierConfig?.maxAlerts ?? 0}`}
```
