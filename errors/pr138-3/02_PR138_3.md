The failure is due to the use of next/dynamic with ssr: false inside a Server Component in app/(dashboard)/charts/[symbol]/[timeframe]/page.tsx. In Next.js, dynamic imports with ssr: false are not allowed in Server Components. You must move the dynamic import into a Client Component.

Solution:

1. Create a new Client Component, for example, app/components/charts/trading-chart-client.tsx:

```tsx
'use client';

import nextDynamic from 'next/dynamic';

const TradingChart = nextDynamic(
  () =>
    import('@/components/charts/trading-chart').then((mod) => mod.TradingChart),
  {
    ssr: false,
    loading: () => (
      <div className="flex h-[600px] items-center justify-center rounded-lg bg-[#1e222d]">
        <div className="text-center">
          <div className="mb-3 inline-block h-8 w-8 animate-spin rounded-full border-2 border-white border-t-transparent"></div>
          <p className="text-[#d1d4dc]">Loading chart...</p>
        </div>
      </div>
    ),
  }
);

export default TradingChart;
```

2. In your page.tsx file, replace the next/dynamic import with the new Client Component:

```tsx
// Remove the existing nextDynamic import and TradingChart definition

import TradingChart from '@/components/charts/trading-chart-client'; // adjust the import path as needed
```

This change ensures the dynamic chart is rendered in a Client Component, resolving the build error. See the code: [page.tsx (dd9acd4e)](<https://github.com/ripper7375/trading-alerts-saas-public/blob/dd9acd4ed099d77c87c6a8f3ed311b127cbcb24c/app/(dashboard)/charts/[symbol]/[timeframe]/page.tsx>).
